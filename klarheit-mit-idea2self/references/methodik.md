<design_thinking>
Der Idea2Self-Prozess adaptiert Design Thinking fuer Einzelpersonen:

| Klassische Phase | Idea2Self-Adaption |
|-----------------|-------------------|
| Empathize | Persona (verhaltensbasiert) + Strasseninterviews |
| Define | Problem-Statement (HMW-Format) + Value Proposition |
| Ideate | KI-gestuetzte Ideen-Werkstatt (5 Vorschlaege) |
| Prototype | Prototyp-Konzept (Konzept, nicht Bau) |
| Test | Experiment-Card + reale Interviews |
</design_thinking>

<lean_startup>
Build-Measure-Learn-Zyklus im Miniformat:

- Hypothese: Explizit formuliert in Experiment-Card
- Experiment: Minimum Viable Experiment (MVE), nicht MVP
- Metriken: Entscheidungsschwelle VOR dem Experiment definieren
- Entscheidung: Weitermachen / Aendern / Lassen

Evidenz-Abstufungen:
- Stark: Validierte Daten, Nutzertests, quantitative Analysen
- Mittel: Qualitative Interviews, Expertenschaetzungen (= Idea2Self-Level)
- Schwach: Bauchgefuehl, unvalidierte Annahmen -> als Experiment dokumentieren
</lean_startup>

<woop>
WOOP (Wish-Outcome-Obstacle-Plan, Gabriele Oettingen):
Evidenzbasierte Selbstregulationsmethode fuer persoenliche Umsetzungshuerden.

Im Action-Plan integriert (nicht separate Uebung):
1. Wish: Umsetzungsziel (konkreter als Vision)
2. Outcome: Bestes Ergebnis wenn alles klappt
3. Obstacle: Persoenliche innere Huerden (nicht externe Risiken)
4. Plan: Wenn-Dann-Plaene fuer jedes Hindernis

Persoenliche Hindernisse sind bei Nebenprojekten haeufiger Grund fuer Scheitern als externe Risiken. Deshalb WOOP im Action-Plan, nicht als Add-on.
</woop>

<methoden_mapping>
| Klassische Methode | Idea2Self-Adaption | Begruendung |
|-------------------|-------------------|-------------|
| Problem-Statement (JTBD/6W/HMW) | Workflow problem-statement | Value Proposition subsumiert |
| Persona / ICP | Workflow persona | Verhaltensbasiert, nicht demografisch |
| Lean Risikohypothesen | Workflow experiment-card | Hypothesen + Metriken integriert |
| Build-Measure-Learn | Interviews + Debrief + Iteration | Ein Zyklus innerhalb des Prozesses |
| MVP / Lo-Fi Prototyping | Workflow prototyp-konzept | Konzept, kein funktionaler Prototyp |
| Sprint Retrospektive | Workflow review-iterate | Erkenntnisse zurueckfuehren |
</methoden_mapping>

<kernprinzipien_umsetzung>

|---------|-----------|

| Schnelles Lernen | Interviews als Minimum Viable Research |
